Positioning

ทำไมเราถึงชนะเจ้าอื่น

จากการวิเคราะห์ audit deck ของเอเจนซี่จริง (ตัวอย่างใน EXsam/ — proposal 25 หน้าของ Relevant Audience) และเครื่องมือในตลาด เราพบช่องว่างที่ยังไม่มีใครทำครบสี่ข้อนี้ในผลิตภัณฑ์เดียว

1 AI Auto-Fix — จุดขายอันดับหนึ่ง

เครื่องมือทุกเจ้า (Ahrefs, SEMrush, Sitebulb) จบที่ "รายการปัญหา" แล้วปล่อยให้ลูกค้าไปจ้างคนแก้เอง เราไปต่ออีกขั้น: AI สร้างโค้ดที่แก้แล้วให้เลย — meta title ใหม่, JSON-LD schema ครบชุด, canonical, hreflang, alt text, robots.txt ที่ถูกต้อง — พร้อมหน้า preview เทียบก่อน/หลัง และดาวน์โหลดไปวางได้ทันที

2 GEO — ความพร้อมยุค AI Search

การค้นหากำลังย้ายไป ChatGPT, Google AI Overview, Perplexity แต่เครื่องมือเก่ายังตรวจแค่ Google เราตรวจ: แบรนด์ถูก AI อ้างถึงไหม, เนื้อหา crawl ได้โดย AI bot หรือเปล่า (GPTBot, ClaudeBot), มี llms.txt ไหม, schema ครบสำหรับ AI ดึงคำตอบหรือไม่ — นี่คือสิ่งที่เอเจนซี่เพิ่งเริ่มขายในราคาแพง

3 หลักฐานระดับวิศวกร — Raw vs Rendered

ตรวจสองรอบ — HTML ดิบ (สิ่งที่ bot เห็นครั้งแรก) เทียบกับหน้าเว็บหลัง render ด้วย headless Chrome แล้วแสดงความต่างให้เห็นชัดว่าอะไร "หายไป" สำหรับ Googlebot และ AI bot (เคส Nuxt SPA แบบ ShareInvestor คือตัวอย่างคลาสสิก: H1 และเนื้อหาทั้งหมดมองไม่เห็นใน HTML ดิบ)

4 รายงานภาษาไทย ขายต่อได้ (White-label)

ผลตรวจออกมาเป็น report deck ระดับ proposal เอเจนซี่ — สรุปผู้บริหาร, ตารางเทียบคู่แข่ง, roadmap — ภาษาไทยเต็มรูปแบบ ใส่โลโก้ลูกค้าหรือเอเจนซี่ได้ ทำให้เอเจนซี่ไทยซื้อไปใช้ออก proposal ของตัวเองได้ทันที

Product Modules

หกโมดูลหลักของระบบ

ออกแบบตามโครงของ audit เอเจนซี่จริง แต่ทำให้อัตโนมัติและลึกกว่าในทุกหมวด

1 Deep Crawler

  • Crawl สองโหมด: HTML ดิบ + rendered (Playwright / headless Chrome)
  • เทียบ diff: title, H1, เนื้อหา, ลิงก์ ที่หายไปใน raw HTML
  • ไล่ตาม sitemap + internal links สูงสุดตามแพ็กเกจ
  • ตรวจ HTTP status ทุกหน้า, redirect chain, CDN error
  • เคารพ robots.txt และจำกัดความเร็ว ปลอดภัยต่อเว็บลูกค้า

2 Technical SEO — 150+ จุดตรวจ

  • Meta title/description (ความยาว, คีย์เวิร์ด, ซ้ำ)
  • โครงสร้าง H1–H6, canonical, hreflang, OG/Twitter tags
  • Schema JSON-LD: Organization, Service, FAQ, Breadcrumb
  • robots.txt, XML sitemap, indexability ราย URL
  • Core Web Vitals (LCP/INP/CLS) ผ่าน PageSpeed API
  • SSL/HSTS, security headers, mobile-friendly, alt text
  • ผลออกเป็น Pass / Warn / Fail พร้อมระดับความสำคัญ

3 GEO — AI Search Readiness

  • ตรวจว่า GPTBot, ClaudeBot, PerplexityBot ถูกบล็อกหรือไม่
  • ทดสอบถาม ChatGPT/Claude/Perplexity จริง — แบรนด์ถูกอ้างถึงหรือไม่
  • Google AI Overview visibility สำหรับคีย์เวิร์ดเป้าหมาย
  • llms.txt, Q&A content blocks, direct-answer structure
  • สัญญาณ E-E-A-T: author schema, team bio, expert quotes

4 Keyword & Competitor Intel

  • ดึง volume / KD / CPC คีย์เวิร์ดไทยและอังกฤษ (DataForSEO API)
  • หา keyword gap: คู่แข่งติดอันดับอะไรที่เราไม่ติด
  • ตารางเทียบ DR, traffic, top keywords แบบหน้า "vs Optiwise"
  • วิเคราะห์หน้า winning ของคู่แข่ง พร้อมแผนแย่งกลับ (AI สรุป)

5 Backlink & Anchor Health

  • โปรไฟล์ backlink: refdomains และการกระจาย anchor text
  • AI จัดกลุ่ม anchor เป็น Healthy / Review / Toxic / Owned
  • ตรวจจับ paid-link footprint และ negative SEO
  • สร้างไฟล์ disavow ให้อัตโนมัติ พร้อมคำอธิบาย

6 AI Auto-Fix Engine

  • AI อ่านปัญหาและบริบทธุรกิจ แล้วสร้างโค้ดแก้จริง
  • Meta/title ใหม่รายหน้า (ไทย+อังกฤษ), JSON-LD ครบชุด, canonical, robots.txt, sitemap, hreflang
  • Preview เทียบก่อน/หลัง พร้อมคะแนนที่จะดีขึ้น
  • ส่งออก: ไฟล์ patch, snippet สำหรับ CMS, หรือ PR ผ่าน GitHub/FTP (เฟสถัดไป)
  • โหมด rewrite เนื้อหาให้ตอบ AI Search (Q&A blocks)

Architecture & Stack

สถาปัตยกรรมระบบ

Tech Stack ที่แนะนำ

  • Frontend: Next.js + Tailwind — dashboard, report viewer, preview ก่อน/หลัง
  • Backend API: Node.js + PostgreSQL
  • Crawler worker: Playwright (headless Chrome) + คิวงานด้วย BullMQ/Redis — แยก process ไม่ให้งาน crawl หนักดึง API ล่ม
  • AI: Claude API — วิเคราะห์, เขียนรายงานไทย, สร้างโค้ดแก้ (Sonnet สำหรับงาน volume, Opus/Fable สำหรับ Auto-Fix)
  • ข้อมูลภายนอก: DataForSEO (keyword/backlink/SERP — ถูกกว่า Ahrefs API มาก), Google PageSpeed API (ฟรี), Search Console OAuth
  • PDF: Playwright print-to-PDF จาก HTML template เดียวกับ report
  • Deploy: VPS Hostinger ที่มีอยู่ (Docker Compose) — เริ่มได้โดยไม่มีต้นทุน cloud เพิ่ม

Pipeline การตรวจหนึ่งครั้ง

  • Stage 1 — Fetch: robots.txt → sitemap → crawl ดิบ → crawl rendered (คู่ขนาน จำกัดหน้าตามแพ็กเกจ)
  • Stage 2 — Rule engine: 150+ checks เป็นโค้ด deterministic (เร็ว ฟรี เที่ยงตรง) — ไม่ใช้ AI ตรวจ เพื่อให้ผลนิ่งและต้นทุนต่ำ
  • Stage 3 — Enrich: ดึง keyword/backlink/CWV จาก API ภายนอก
  • Stage 4 — AI Layer: AI รับผลตรวจ + บริบท ไปตีความ จัดลำดับ เขียนรายงาน — AI ทำเฉพาะส่วนที่ AI เก่ง
  • Stage 5 — Render: ออก report และเก็บ history ไว้เทียบครั้งถัดไป

หลักคิดสำคัญ: rule engine ตรวจ / AI ตีความและแก้ — แยกหน้าที่ชัด ผลตรวจเชื่อถือได้เต็มร้อย และค่า AI ต่อหนึ่ง audit ต่ำมาก (ประมาณ 10–30 บาท)

ข้อควรระวังที่ต้องวางแผนแต่แรก: (1) ต้นทุน DataForSEO คิดต่อ request — ต้องทำ cache และจำกัดตามแพ็กเกจ (2) การทดสอบ citation บน ChatGPT/Perplexity ต้องใช้ API ทางการเท่านั้น (3) Auto-Fix ที่เขียนไฟล์ใส่เว็บลูกค้าโดยตรง ต้องมีระบบ backup/rollback เสมอ — เฟสแรกให้ "ดาวน์โหลดไฟล์แก้" ปลอดภัยกว่า "apply อัตโนมัติ"

  • Web report พร้อมคะแนนรวม 0–100 รายหมวด

สัปดาห์ 4–6
GEO + คู่แข่ง

Phase 3

AI Auto-Fix
