Phase 2: DataForSEO Integration

⏱️ 3-4 days Priority: 🟠 HIGH
  • DataForSEO SDK Setup
    • npm install @dataforseo/client
    • ตั้ง DATASEO_API_KEY ใน .env
    • Test fetch keyword data (3 keywords)
  • Keyword Opportunities Module
    • lib/keywords.js: ดึง vol/KD/CPC ของ target keywords
    • เพิ่ม projection: traffic = vol × (CTR based on #1-5)
    • ใส่ audit.keywords[] + present.js สไลด์
  • Backlink Health + Anchor Analysis
    • DataForSEO SERP task: fetch competitor backlinks
    • Identify toxic anchors (exact-match spam patterns)
    • Disavow recommendations
  • CPC Monetization Estimate
    • traffic × avg CPC = monthly revenue potential
    • Show ใน projection slide
✓ Success Metrics:
- Audit ได้ keyword vol/KD/CPC จริง
- Projection traffic estimate ≥ 80% ของเลิกค้า expectation (calibrate ด้วย GSC data)
- Backlink recommendations ≥ 3 ข้อต่อ audit
Dependency: DataForSEO API key (ต้องติดต่อพวกเขา + ค่า $100-200/month)
Blocks: Phase 3 (GSC) เพื่อ calibrate projection accuracy

Phase 3: GSC OAuth + Real Traffic Data

⏱️ 2-3 days Priority: 🟡 MEDIUM
  • Google OAuth2 Flow
    • Register OAuth app (google-auth-library-nodejs)
    • POST /oauth/gsc-connect → browser consent → redirect callback
    • Store refresh token securely (data/tokens/ encrypted)
  • GSC Data Fetch
    • Fetch real impressions/clicks/CTR/position (last 28 days)
    • Merge ด้วย audit findings (keywords in top 20)
    • Show "opportunity" = high impression + low CTR
  • Traffic Projection Calibration
    • เทียบ predicted vs real GSC data
    • Adjust formula (learning rate)
✓ Success Metrics:
- GSC auth flow 100% working
- Projection error margin < 20% vs real GSC
- "Opportunity" keywords identified ≥ 5
Dependency: Phase 2 (DataForSEO keyword data)
Needs: Real GSC data (customer account)

⚠️ Known Risks & Blockers

🔴 Critical Blockers

WAF Blocks Crawler (shareinvestor.com)
Upstream WAF บล็อก external requests → เลือกลูกค้าใหม่สำหรับ demo หรือให้ลูกค้า whitelist IP
DataForSEO API Key ต้องซื้อ
ยังไม่ซื้อ → Phase 2 ต้องรอ; ทดแทนชั่วคราวกับ SemRush/Ahrefs trial account หรือ fallback estimates
Google OAuth Approval Timeline
Google verification อาจใช้ 1-2 สัปดาห์ → ต้อง request review ก่อน Phase 3

🟠 Medium Risks

Playwright Rendering Performance
Large sites (100+ pages) = slow crawl (30+ min) → ต้อง set max pages limit หรือ parallel workers
CWV API Rate Limiting (Google PageSpeed)
150 requests/minute → site ที่ 100+ pages อาจhit limit → ต้อง queue + backoff
Fix Accuracy on Complex Sites
Demo site (4 pages, ง่าย) = 94% fixed; production site อาจต่ำกว่า → ต้อง increase test coverage

🟡 Low Risks (Known)

Node.js Fetch Sandbox (external sites)
Verify.mjs ต้องใช้ localhost — ขึ้นขึ้นอยู่ environment setup; workaround: curl wrapper
Cheerio vs Playwright Mismatch
SPA shell บางทีไม่ render ถูก → สร้าง fallback checklist manual
Storage (data/audits/ unlimited growth)
ต้อง implement archive/cleanup policy ก่อน production (>1000 audits)
