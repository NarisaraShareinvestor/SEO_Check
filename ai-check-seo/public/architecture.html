System Architecture · Engineering Notes

สถาปัตยกรรมระบบ
และ แนวคิดเบื้องหลัง

เอกสารนี้อธิบายว่าระบบประกอบด้วยอะไรบ้าง — ฝั่ง Backend, Frontend, การไหลของข้อมูล และแนวคิดทางวิศวกรรมที่ทำให้ผลการตรวจ "แม่นยำ ทำซ้ำได้ และตรวจสอบย้อนกลับได้"

01

ภาพรวมสถาปัตยกรรม

4 ชั้น + บริการภายนอก
เบราว์เซอร์เรียก API → Engine ตรวจ (เรียกบริการภายนอกตามต้องการ) → เก็บผลเป็น JSON → ส่งกลับให้หน้าเว็บแสดง
02

แนวคิดหลักทางวิศวกรรม

การตัดสินใจที่ทำให้ผลเชื่อถือได้
Raw-first

วัดจาก HTML ดิบ

วิเคราะห์สิ่งที่ Googlebot รอบแรกและ AI crawler อ่านได้ก่อนรัน JavaScript — เพราะ AI bot ส่วนใหญ่ไม่ประมวลผล JS

Two-wave

เก็บทั้ง raw และ rendered

crawler เปิดหน้าด้วย headless Chrome เก็บทั้งโค้ดดิบและหน้าหลัง render แล้วเทียบกัน ตามพฤติกรรม two-wave indexing

Deterministic

กฎตายตัว ผลนิ่ง

หัวใจคือ rule engine ที่ไม่สุ่ม (ไม่ใช่ AI ตัดสิน) — เว็บเดิมได้คะแนนเดิม จึงเทียบก่อน/หลังแก้ได้จริง

AI as a layer

AI เป็นชั้นเสริม มี guardrail

ใช้ AI เฉพาะการประเมินเชิงคุณภาพ + มีด่านตรวจกัน hallucination (ตัด finding ที่อ้างอิง check ที่ไม่มีจริง)

Cross-check

ไม่ใช้ engine ตัดสิน engine

เทียบผลกับ Google Lighthouse เป็นเกณฑ์อิสระ คำนวณ Precision/Recall รายเว็บ เพื่อจับกรณีตรวจผิด

Resilient

ทนต่อความไม่เสถียร

retry ตาม Retry-After, แยก timeout (สรุปไม่ได้) ออกจากปัญหาจริง, ค่าที่ผันผวนไม่กระทบคะแนน

03

Backend

Node.js + Express · ตรรกะทั้งหมดอยู่ใน lib/
โมดูลหน้าที่
server.jsExpress API + คิวงานเบื้องหลัง + บันทึก/อ่านผลเป็นไฟล์ JSON + เส้นทางรายงาน/PDF
crawler.jsดึง HTML ดิบ (retry, จัดการ 429/redirect/TLS), rendered crawl ด้วย Playwright สำหรับ SPA, ตรวจรูปที่มองเห็นจริง, robots/sitemap, broken links, การตรวจแบบสุภาพ
checks.jsRule engine — กฎ deterministic ~75 จุด แบ่ง 9 หมวด (มี ENGINE_VERSION สำหรับเทียบเวอร์ชัน)
scorer.jsคำนวณคะแนน 0–100 ถ่วงน้ำหนักตามความรุนแรง × หมวด → เกรด A–F
geo.jsตรวจความพร้อมด้าน AI Search — robots สำหรับ AI bot, SPA risk, FAQ schema, E-E-A-T, llms.txt
verify.js / psi.jsดึง Google Lighthouse / PageSpeed มาตรวจทาน + คำนวณ Precision/Recall + ระดับความเชื่อมั่นรายข้อ
ai.jsวิเคราะห์เชิงคุณภาพด้วย OpenAI + guardrail กัน hallucination
schema-validate.jsตรวจ JSON-LD / schema.org ตามเกณฑ์ Google
report-exec · report-sales · reportสร้างรายงาน 3 แบบ (ผู้บริหาร / เซลส์ / เทคนิค) เป็น HTML → PDF/PPTX
autofix · pagefixสร้างหน้าเว็บ "ฉบับแก้แล้ว" ให้ dev นำไปใช้
clickup.jsส่ง Action Item พร้อมโค้ดแก้ไปยัง ClickUp ให้ทีม dev
04

Frontend

SPA แบบ vanilla — ไม่มี build step ไม่พึ่ง framework
index.html + app.js

Single-page app

แถบเมนูด้านข้าง (ตรวจเว็บ · ประวัติ · คุณภาพการตรวจ · Dashboard · หลักการให้คะแนน) สลับมุมมองด้วย JS · เรียก API แล้ว poll สถานะจนตรวจเสร็จ · เรนเดอร์ผล/กราฟ/ตารางคุณภาพ

รายงาน (server-rendered)

HTML ที่พิมพ์/ส่งออกได้

รายงานสร้างฝั่งเซิร์ฟเวอร์เป็น HTML สไลด์ 16:9 · ดาวน์โหลด PDF (headless Chrome) · รองรับ white-label (โลโก้/สีแบรนด์)

เลือกใช้ vanilla JS โดยตั้งใจ — ไม่มีขั้นตอน build, deploy ง่าย, โหลดเร็ว และไม่มี dependency frontend ให้ดูแล
05

การไหลของข้อมูล (1 รอบการตรวจ)

  1. รับคำขอ
    หน้าเว็บส่ง POST /api/audit (URL + จำนวนหน้า) → เซิร์ฟเวอร์สร้าง job แล้วทำงานเบื้องหลัง คืน id ทันที
  2. Crawl
    ดึง HTML ดิบทุกหน้า + ถ้าเป็น SPA เปิด headless Chrome เก็บหน้า render · ตรวจ robots/sitemap/broken links · เก็บรูปที่มองเห็นจริง
  3. ตรวจด้วยกฎ
    runChecks (deterministic ~75 จุด) + runGeoChecks (ความพร้อม AI) → ได้รายการผลพร้อมความรุนแรง/สถานะ
  4. ให้คะแนน
    scoreAudit ถ่วงน้ำหนัก 9 หมวด → คะแนน 0–100 + เกรด
  5. วิเคราะห์ AI
    ai.js สรุปประเด็นสำคัญเชิงธุรกิจ ผ่าน guardrail ก่อนแสดง
  6. ตรวจทาน
    verify.js เทียบกับ Google Lighthouse (ขนานกัน) → ติดธง "ควรรีวิว" ถ้าผลต่างจาก Google ในข้อเท็จจริง
  7. บันทึก + ส่งกลับ
    เก็บผลทั้งหมดเป็น data/audits/<id>.json · หน้าเว็บ poll GET /api/audit/:id จนเสร็จแล้วเรนเดอร์
06

Tech Stack

Node.js ≥18 (runtime) Express (web server / API) cheerio (parse HTML ดิบ) Playwright / Chromium (render SPA + สร้าง PDF) OpenAI API (วิเคราะห์เชิงคุณภาพ) Google PageSpeed / Lighthouse (ตรวจทาน + CWV) VPS + pm2 + nginx (deploy)
เก็บผลเป็นไฟล์ JSON (ไม่ต้องมีฐานข้อมูล) — เบา ตรวจสอบย้อนกลับได้ และทุกการตรวจเป็นหลักฐานในตัวเอง · deploy ด้วย git → ดึงลง VPS → pm2 restart
