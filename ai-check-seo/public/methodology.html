03

หลักการพื้นฐาน

เหตุผลเบื้องหลังวิธีวัด
Raw-first

วัดจาก HTML ดิบ

ประเมินสิ่งที่ Googlebot รอบแรกและ AI crawler อ่านได้ก่อนรัน JavaScript เพราะ crawler ส่วนใหญ่ไม่ประมวลผล JS

Two-wave

เทียบหน้า render จริง

เปิดด้วย headless Chrome เทียบ raw กับ rendered ตามพฤติกรรม two-wave indexing ที่ Google ใช้

Deterministic

ผลนิ่ง ทำซ้ำได้

ใช้กฎตายตัวไม่สุ่ม เว็บเดิมได้คะแนนเดิม จึงเปรียบเทียบก่อน/หลังแก้ได้อย่างมีความหมาย

Mobile-first

วัดแบบมือถือ

ประเมินรูปและ viewport แบบที่ผู้ใช้มือถือเห็นจริง (กระตุ้น lazy-load ก่อนวัด) ตรงกับที่ Google ใช้ index

Generative

ความพร้อมด้าน AI

ตรวจว่า robots.txt อนุญาต AI crawler ที่ระบุชื่อหรือไม่ และเนื้อหาอยู่ในรูปที่ AI ดึงไปใช้ได้ (ไม่พึ่ง JS ล้วน)

Verified

ตรวจทานกับ Google

เทียบผลกับ Google Lighthouse เป็นเกณฑ์อิสระ พร้อมวัด Precision / Recall รายเว็บ เพื่อกันการตรวจผิด

04

วิธีคิดคะแนน

สามชั้น จาก check เดี่ยวสู่คะแนนรวม
ชั้น 1 · check
หักคะแนน = น้ำหนักความรุนแรง × โทษตามสถานะ
ชั้น 2 · หมวด
คะแนนหมวด = ( 1 − Σ หักคะแนน ÷ Σ น้ำหนักเต็ม ) × 100
ชั้น 3 · รวม
คะแนนรวม = Σ ( คะแนนหมวด × น้ำหนักหมวด ) ÷ 100
ตัวอย่างหมวดรูปภาพ — img-alt (รุนแรงปานกลาง = 5, สถานะ Fail → โทษ 100%), img-lazy (เล็กน้อย = 2, Warn → 45%), img-dimensions (เล็กน้อย = 2, Pass → 0%)
น้ำหนักเต็ม = 5 + 2 + 2 = 9 · หักจริง = (5 × 1.0) + (2 × 0.45) + 0 = 5.9 · คะแนนหมวด = (1 − 5.9 ÷ 9) × 100 = 34
