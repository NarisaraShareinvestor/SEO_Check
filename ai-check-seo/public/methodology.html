02

ภาพรวมวิธีการวัด

จากการเก็บข้อมูลถึงคะแนนและการตรวจทาน
เก็บทั้ง HTML ดิบและหน้าที่ render แล้ว → ประเมินด้วยกฎตายตัว 9 หมวด → ถ่วงน้ำหนักเป็นคะแนน → ตรวจทานกับ Google Lighthouse
03

หลักการพื้นฐาน

เหตุผลเบื้องหลังวิธีวัด
Raw-first

วัดจาก HTML ดิบ

ประเมินสิ่งที่ Googlebot รอบแรกและ AI crawler อ่านได้ก่อนรัน JavaScript เพราะ crawler ส่วนใหญ่ไม่ประมวลผล JS

Two-wave

เทียบหน้า render จริง

เปิดด้วย headless Chrome เทียบ raw กับ rendered ตามพฤติกรรม two-wave indexing ที่ Google ใช้

Deterministic

ผลนิ่ง ทำซ้ำได้

ใช้กฎตายตัวไม่สุ่ม เว็บเดิมได้คะแนนเดิม จึงเปรียบเทียบก่อน/หลังแก้ได้อย่างมีความหมาย

Mobile-first

วัดแบบมือถือ

ประเมินรูปและ viewport แบบที่ผู้ใช้มือถือเห็นจริง (กระตุ้น lazy-load ก่อนวัด) ตรงกับที่ Google ใช้ index

Generative

ความพร้อมด้าน AI

ตรวจว่า robots.txt อนุญาต AI crawler ที่ระบุชื่อหรือไม่ และเนื้อหาอยู่ในรูปที่ AI ดึงไปใช้ได้ (ไม่พึ่ง JS ล้วน)

Verified

ตรวจทานกับ Google

เทียบผลกับ Google Lighthouse เป็นเกณฑ์อิสระ พร้อมวัด Precision / Recall รายเว็บ เพื่อกันการตรวจผิด

04

วิธีคิดคะแนน

สามชั้น จาก check เดี่ยวสู่คะแนนรวม
ชั้น 1 · check
หักคะแนน = น้ำหนักความรุนแรง × โทษตามสถานะ
ชั้น 2 · หมวด
คะแนนหมวด = ( 1 − Σ หักคะแนน ÷ Σ น้ำหนักเต็ม ) × 100
ชั้น 3 · รวม
คะแนนรวม = Σ ( คะแนนหมวด × น้ำหนักหมวด ) ÷ 100
ตัวอย่างหมวดรูปภาพ — img-alt (รุนแรงปานกลาง = 5, สถานะ Fail → โทษ 100%), img-lazy (เล็กน้อย = 2, Warn → 45%), img-dimensions (เล็กน้อย = 2, Pass → 0%)
น้ำหนักเต็ม = 5 + 2 + 2 = 9 · หักจริง = (5 × 1.0) + (2 × 0.45) + 0 = 5.9 · คะแนนหมวด = (1 − 5.9 ÷ 9) × 100 = 34

ความรุนแรง × สถานะ

แต่ละผลหักคะแนนเท่าไร
ระดับความรุนแรงน้ำหนัก
High — ร้ายแรง10
Medium — ปานกลาง5
Low — เล็กน้อย2
สถานะผลตรวจโทษ
Fail — เสียจริง100%
Warn — ควรปรับ45%
Pass — ผ่าน0%
Info — ข้อมูล0%
สถานะ Info ไม่กระทบคะแนน ใช้กับสิ่งที่สรุปไม่ได้ในรอบนั้น เช่น HTTP 429 (rate-limit), 5xx ชั่วคราว หรือ render ไม่สำเร็จ — เพื่อกันคะแนนแกว่งจากปัจจัยเครือข่ายที่ไม่เกี่ยวกับคุณภาพเว็บ
07

มาตรฐานอ้างอิง

ทุกการวัดอิงเอกสารทางการ ไม่ใช่ความเห็นส่วนตัว
กรณีตัวอย่าง — ทำไมวัด H1 แบบนี้
วัดอะไร
แต่ละหน้าควรมี H1 หนึ่งตัว ไม่ว่างเปล่า และไม่ถูกซ่อนด้วย CSS — ตรวจทั้งใน HTML ดิบและหน้า render
อิงมาตรฐาน
HTML Living Standard (WHATWG) §4.3 ว่าด้วยโครงสร้างหัวข้อ · WCAG 2.2 เกณฑ์ 1.3.1 “Info and Relationships” (โครงสร้างต้องอ่านได้ด้วยโปรแกรม) · Google Search Central — แนะนำหัวข้อที่สื่อความหมายเพื่อช่วยเข้าใจหน้า และมีผลต่อการดึง featured snippet
เหตุผล
H1 คือหัวเรื่องหลักที่บอกทั้งผู้ใช้ เครื่องช่วยอ่าน และเครื่องมือค้นหาว่าหน้านี้เกี่ยวกับอะไร — หายหรือซ่อน = โครงสร้างกำกวม, ซ้ำหลายตัว = สับสนว่าหน้าเน้นอะไร
