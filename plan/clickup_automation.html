Automation: SEO / AI Readiness Report → ClickUp Action Items

ออกแบบระบบอ่านผลตรวจจาก AI SEO Audit Pro แล้วแปลงประเด็นที่พบเป็น Todo / Action Item ใน ClickUp อัตโนมัติ

เอกสารออกแบบ (Design Spec) · เวอร์ชัน 0.1 · 18 มิ.ย. 2569 · สำหรับเสนอที่ประชุม + วางระบบใน ClickUp

เอกสารนี้คือ การออกแบบก่อนพัฒนา — อธิบายว่าจะดึงข้อมูลจาก report ที่ระบบเราผลิตอยู่แล้ว (audit JSON ใน data/audits/) มาแปลงเป็นงานใน ClickUp อย่างไร, แมปฟิลด์อะไรไปอะไร, และเดินงานเป็นเฟสไหน — เพื่อให้เห็นภาพตรงกันก่อนลงมือเขียนโค้ด

1 เป้าหมาย & ปัญหาที่แก้

ทุกวันนี้พอตรวจเว็บเสร็จ ได้รายงานปัญหา 30–40 ข้อต่อเว็บ ทีมต้อง อ่าน report เอง แล้วเปิด Todo ใน ClickUp ทีละข้อ — เสียเวลา, ตกหล่น, และคัดลอกผิดมือ เมื่อมีหลายเว็บ/หลายรอบยิ่งไม่ไหว

เป้าหมาย: เมื่อมี report ใหม่ ระบบ สร้าง Action Item ใน ClickUp ให้อัตโนมัติ — จัด Priority / Category / Team ให้ครบ, ส่งเข้า Project ปลายทางที่ถูกต้อง (เช่น เว็บลูกค้า PTG → Project PTG), และ นำ workflow เดียวไปใช้ซ้ำกับทุกเว็บได้

2 ภาพรวมการทำงาน (Architecture)

1
Audit เสร็จระบบตรวจเว็บ → ได้ audit JSON (มี checks/score/GEO อยู่แล้ว)
2
Transformerกรองเฉพาะปัญหา (fail/warn) → แปลงเป็น Action Item ตามกฎแมป
3
Routerดูว่าเว็บนี้เป็นของลูกค้าไหน → เลือก ClickUp List ปลายทาง
4
ClickUp APIสร้าง/อัปเดต Task (เว็บ) + Subtask (ปัญหา) แบบกันซ้ำ
5
ทีมรับงานเห็นงานพร้อม URL/วิธีแก้/Priority/Owner — ลงมือต่อได้ทันที

จุดแข็ง: ระบบเรามีข้อมูลโครงสร้างพร้อมอยู่แล้ว — แต่ละ check มี id / category / severity / status / title / detail / recommendation / pages / fixable ฉะนั้น automation นี้คือ "ชั้นแปลง + ส่ง" บางๆ ไม่ต้องตีความ report ใหม่

4 กฎการแมป (Mapping Rules)

4.1 Severity → ClickUp Priority

ในระบบเราเงื่อนไขClickUp PriorityDue (ดีฟอลต์)
severity:high + failปัญหาร้ายแรงUrgent3 วัน
high+warn หรือ med+failควรรีบแก้High7 วัน
med + warnควรปรับปรุงNormal14 วัน
lowเล็กน้อยLow30 วัน

ปัญหาที่อยู่ใน analysis.topPriorities (AI จัดอันดับผลกระทบธุรกิจ) ดัน Priority ขึ้น 1 ขั้น + ขึ้นต้น list

4.2 Category (ของเรา) → กลุ่มงาน / ทีมที่รับผิดชอบ

category (เรา)กลุ่มงาน ClickUpทีมตัวอย่าง check
indexIndexingTechnical SEO / Devcanonical, sitemap, robots, noindex
schemaSchemaTechnical SEO / Devjsonld-missing, schema-incomplete, og-tags
performancePerformanceDev / DevOpscwv-score, ttfb-slow, compression
onpageContent / SEOContent / SEOtitle, meta description, thin content
geoAI ReadinessSEO + Devgeo-faq-schema, llms.txt, geo-entity
links / images / rendering / securityTechnical SEODevbroken-links, img-alt, spa-shell, https

แมปนี้เก็บเป็น config (ตารางเดียว) — แก้ทีหลังได้ไม่ต้องแตะโค้ด · 9 category ของเราครอบคลุมกลุ่มที่ขอ (SEO, Technical SEO, Content, AI Readiness, Schema, Indexing, Performance) ครบ

4.3 หน้าตา Subtask (Description Template)

# {check.title}  — เช่น "ไม่มี Structured Data (JSON-LD) เลย"

🔍 ตรวจเจอจริง   : {check.detail}
💡 แนวทางแก้      : {check.recommendation}
📄 หน้าที่กระทบ   : {affectedCount} หน้า
   - https://site/a   - https://site/b  (สูงสุด 10 + "และอีก N")
🔗 รายงานเต็ม     : https://seo.ohmai.me/report-sale/{auditId}
🏷 หมวด          : {กลุ่มงาน}   |  Owner: {ทีม}
issue-key: {auditId}:{check.id}   ← ฟิลด์ซ่อน ไว้กันซ้ำ/re-scan

5 กันซ้ำ & Re-scan (Idempotency)

หัวใจที่ทำให้ใช้ซ้ำได้และต่อยอด Auto-Recheck — ทุก subtask ฝัง issue-key = {auditId หรือ domain}:{check.id} ใน 1 เว็บ = 1 parent task พอตรวจรอบใหม่:

สถานะ check รอบใหม่การกระทำใน ClickUp
เคย fail/warn → ตอนนี้ passปิด subtask เป็น Complete ✅ (= แก้แล้ว, ผูก Auto-Recheck)
ยัง fail/warn (มีอยู่แล้ว)อัปเดต detail/affectedCount/Priority — ไม่สร้างซ้ำ
ปัญหาใหม่ (ไม่เคยมี)สร้าง subtask ใหม่

จับคู่ด้วย issue-key (ค้นใน parent task ก่อนสร้าง) → รันกี่รอบก็ไม่เกิด Todo ซ้ำ · ตรงกับ Acceptance "นำ workflow ไปใช้ซ้ำได้"

6 ส่งเข้า Project ปลายทาง (Routing)

เว็บแต่ละเว็บส่งเข้า ClickUp List ของลูกค้าที่ถูกต้องอัตโนมัติ — เก็บเป็น config data/clickup-routing.json

{
  "ptgenergy.co.th": { "listId": "901xxx...", "team": "PTG", "defaultAssignee": "u123" },
  "richsport.co.th": { "listId": "902xxx...", "team": "RichSport" },
  "_default":        { "listId": "900xxx...", "team": "SEO Inbox" }
}

เช่น report ของเว็บ PTG → สร้าง Action Item ที่ Project PTG · เว็บที่ยังไม่ตั้ง routing → เข้า "SEO Inbox" (ดีฟอลต์) แล้วค่อยย้าย

7 ตัวจุดชนวน (Trigger)

  • ปุ่มในแดชบอร์ด "ส่ง Action Items เข้า ClickUp" บนหน้าผลตรวจ — กดเองเมื่อพร้อม (เฟส 1)
  • อัตโนมัติเมื่อ audit เสร็จ — server.js hook: ถ้าเว็บนี้มี routing ตั้งไว้ → ส่งทันที (เฟส 2)
  • ผูกกับ Watchlist ที่มีอยู่ — ตรวจตามรอบแล้ว sync subtask อัตโนมัติ (เฟส 3)

8 การเชื่อมต่อ ClickUp (API / MCP)

ทางเลือก A — ClickUp REST API v2 แนะนำ (production)

งานEndpoint
สร้าง Task (เว็บ)POST /list/{listId}/task
สร้าง SubtaskPOST /list/{listId}/task + parent
ตั้ง custom fieldPOST /task/{id}/field/{fieldId}
ค้นกันซ้ำGET /list/{listId}/task?...
ปิดงาน (re-scan)PUT /task/{id} status

Auth: Personal API Token หรือ OAuth · รองรับ priority/tags/assignees/due date ครบ

ทางเลือก B — ClickUp MCP prototype / ทดสอบเร็ว

มี ClickUp MCP ที่ต่อกับ Claude ได้เลย — เหมาะ ทดสอบ flow / สร้าง list/field โครงครั้งแรก โดยไม่ต้องเขียน API client

แต่ production ควรเป็น REST API (รันเองบน VPS, ไม่ผูกกับ session) — ใช้ MCP ช่วยตอน setup ครั้งแรกแล้วย้ายเป็น API

POST /api/v2/list/901.../task
{
  "name": "ไม่มี Structured Data (JSON-LD) เลย",
  "parent": "<website-task-id>",

  "tags": ["Schema"],
  "assignees": [123],
  "due_date": 1718800000000,
  "markdown_description": "🔍 ตรวจเจอจริง: 0/30 หน้า ...\n💡 แนวทางแก้: ...",
  "custom_fields": [{ "id":"issue_key", "value":"a1b2:jsonld-missing" }]
}

10 แผนพัฒนา (เป็นเฟส ส่งมอบได้ทีละก้อน)

เฟส 1 — Transformer + ส่งแบบกดเอง~3–4 วัน
  • เขียน lib/clickup.js: แปลง audit → action items (ตารางแมป 4.1/4.2/4.3)
  • ต่อ ClickUp API (token + 1 list ทดสอบ) สร้าง parent task + subtasks
  • ปุ่ม "ส่งเข้า ClickUp" บนหน้าผล + custom field "issue-key"
  • เกณฑ์ผ่าน: กดแล้วเกิด Task เว็บ + subtask ครบทุกปัญหา พร้อม Priority/URL/วิธีแก้
เฟส 2 — Routing + อัตโนมัติ + กันซ้ำ~3 วัน
  • data/clickup-routing.json (domain → list/team) + map หลายลูกค้า (PTG ฯลฯ)
  • กันซ้ำด้วย issue-key (ค้นก่อนสร้าง) — รันซ้ำไม่เกิด Todo ซ้ำ
  • auto-trigger เมื่อ audit เสร็จ (server.js hook)
เฟส 3 — Re-scan Sync + ผูก Auto-Fix~2–3 วัน
  • Re-scan: ปิด subtask ที่แก้แล้ว / อัปเดตที่ยังค้าง / เพิ่มปัญหาใหม่ (ข้อ 5)
  • ผูกกับ Watchlist (ตรวจตามรอบ → sync เอง)
  • ปุ่ม/ลิงก์ AI Auto-Fix จาก subtask → แก้แล้ว recheck → ปิดงานอัตโนมัติ

11 สิ่งที่ต้องขอ / ตัดสินใจก่อนเริ่ม

ข้อมูลที่ต้องขอ:
  • ClickUp API Token (หรือ OAuth) + สิทธิ์เขียนใน Workspace
  • List ID ปลายทางของแต่ละลูกค้า (เช่น Project PTG) + โครง Space/Folder/List ที่ต้องการ
  • Custom Field IDs (SEO Score, GEO Score, Severity, Issue Key ฯลฯ) — หรือให้เราสร้างให้ผ่าน MCP/API
  • รายชื่อสมาชิก/ทีม ใน ClickUp (assignee IDs) สำหรับแมป category → owner
  • ยืนยันชุด Priority/Due date ตามตาราง 4.1 ว่าตรงกับ SLA ทีมไหม
